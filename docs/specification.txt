--------------------------------------------------------------------------------
1. FLAKE INFRASTRUCTURE (flake.nix)
--------------------------------------------------------------------------------
The project uses a Nix Flake for reproducible system deployment and dev shells.

Inputs:
  - nixpkgs: nixos-26.05 branch, primary package set.
  - nixpkgs-legacy: Frozen 24.11 for a single package (promtail; removed from newer
    nixpkgs during the Loki 3 transition). Consumed via overlays/default.nix.
  - home-manager: User environment sync (inline in nixosConfigurations).
  - sops-nix: Declarative secret management for encrypted, committable secrets.
  - rust-overlay: Nightly/Stable Rust toolchains.
  - hyprland: Wayland compositor, pinned to v0.55.4.
  - hypr-config: RomeoCavazza/hyprland-config (flake=false). Source for the
    Hyprland/Waybar/Rofi/foot configs, rendered with palette injection.
  - hyprspace / hyprland-plugins / hyprtasking: flake inputs, follow hyprland.
  - hyprchroma / hypr-canvas: RomeoCavazza forks (flake=false), built as Nix
    derivations under pkgs/hyprland-plugins/.
  - nix-snapd: Canonical Snap daemon implementation for NixOS.

Outputs:
  - nixosConfigurations.legion: primary entry point. System + Home Manager applied atomically.
  - devShell + apps/checks: local quality gate (nixfmt, deadnix, statix, grafana drift check).
  Per-project development uses home-manager packages plus per-project flakes with direnv.

--------------------------------------------------------------------------------
2. SYSTEM LAYER (hosts/legion + profiles)
--------------------------------------------------------------------------------
Entry point: hosts/legion/default.nix imports hardware-configuration.nix and
profiles/workstation.nix, which composes the profiles (core, boot, hardware,
services, desktop-hyprland, launcher, observability). Profiles import the domain
modules under modules/{boot,core,desktop,hardware,services,observability}/.

Boot Logic (systemd-boot):
  - configurationLimit = 1: Limits boot menu to 1 NixOS entry. Rollback via
    nixos-rebuild switch --rollback or flake.lock revert + rebuild.
  - Dual Boot: Custom Windows 11 entry pointing to /EFI/Microsoft/Boot/bootmgfw.efi.
  - I2C Support: Explicit loading of i2c-dev/i2c-i801 for hardware sensors.

Nix Runtime:
  - auto-optimise-store = true: Deduplication via hard-linking.
  - sandbox-build-dir = "/build": Bind mount to avoid /tmp bloat during heavy builds.
  - GC Policy: Weekly cleanup of items older than 7 days.

Display & Desktop Environment:
  - Hybrid DM: GDM managing both GNOME and Hyprland.
  - XDG Portal: Dual implementation (hyprland/gtk) for Wayland compatibility.

System Services:
  - Guix: Native Guix daemon integration (from the primary nixpkgs).
  - Pipewire: High-performance audio with ALSA and Pulse layers.
  - Polkit: GNOME auth-agent integrated into systemd user session.
  - Restic backups: Encrypted cloud snapshots to Backblaze B2 via sops-nix.

--------------------------------------------------------------------------------
3. USER ENVIRONMENT (home/tco/default.nix)
--------------------------------------------------------------------------------
Managed via Home-Manager. Imports: packages/, hyprland/, dotfiles.nix, scripts/,
shell.nix, gtk.nix, apps.nix, emacs.nix.

Session & Paths:
  - $PATH injection: .lmstudio/bin, .npm-global/bin, .local/bin (priority order).
  - Wayland Flags: QT_QPA_PLATFORM=wayland, NIXOS_OZONE_WL=1 for Chrome/Electron.

Package Ecosystem:
  - Standard Tools: Bat, Eza, Fzf, Zoxide, Direnv.
  - Creative Tools: OBS Studio, Discord, Spotify.
  - Development: Zed, NeoVim, VSCode (with Nix/Python/Rust extensions).
  - CAD/EDA: KiCad, FreeCAD (→ home/tco/packages/cad.nix).
  - Embedded: Arduino IDE/CLI, esptool, minicom (→ home/tco/packages/embedded.nix).
  - Data: DBeaver, database tooling (→ home/tco/packages/data.nix).

Dotfile Management:
  - Theme: Hyprland/Waybar/Rofi/foot configs are rendered into the store from the
    hyprland-config flake input with palette injection (not raw symlinks). Only
    nvim/doom are out-of-store symlinks to the live checkout for zero-rebuild edits.
  - Antigravity: Multi-binary integration (system-wide vs local shim).
  - SOPS secrets: Encrypted YAML stored in-repo; decrypted locally via Age/SSH.

4. OPTIONAL MODULES (modules/)
--------------------------------------------------------------------------------
Modular toggles for specific workstation workflows.

nvidia-prime.nix:
  - Offload logic: Integrates NVIDIA-DRM and kernel parameters for 3D performance.

virtualisation.nix:
  - Libvirt/KVM enabled.
  - binfmt emulation: Allows running aarch64 (ARM) binaries on x86_64 host.

ollama.nix:
  - Local AI daemon (ollama-cuda). OLLAMA_NUM_CTX=32768, OLLAMA_KEEP_ALIVE=24h,
    OLLAMA_KV_CACHE_TYPE=q8_0.

observability.nix:
  - Prometheus :9090, Node Exporter :9100, NVIDIA GPU Exporter :9835,
    Loki :3100, Grafana listener :3001 (proxied to :3000 by nginx.nix).
  - Promtail runs as raw systemd service (NixOS module conflict workaround),
    binary sourced from pkgs-legacy.promtail via custom overlay.
  - Four dashboards provisioned from /etc/nixos/config/grafana/ (hot-reload
    mutable path): nixos-metrics.json, nix-efficiency.json,
    incident-correlation.json, nixos-compiled.json.
  - Local Continuous Documentation: `grafana-snapshot-sync.timer` (every 6h)
    captures dashboards via headless Playwright/Chrome, writes PNGs to a
    throw-away orphan Git commit, and force-pushes to the remote `snapshots`
    branch (`MIN_CHANGE_PERCENT=0.3` gate). The `snapshots` branch is kept at
    exactly 1 commit depth; the `main` branch history is never touched.
    Dashboard images in README point to raw GitHub URLs on `snapshots`.
  - Additional timers: nix-metrics (15min), hypr-metrics (30s).
  - Log correlation labels: promtail injects `component=display|build|system`.

gdm-wallpaper.nix:
  - Custom NixOS module (not in nixpkgs). Patches gnome-shell-theme.gresource
    by injecting CSS overrides, recompiling with glib-compile-resources.

databases.nix:
  - PostgreSQL 17 + PostGIS, Redis (2GB allkeys-lru, AOF+RDB), Qdrant :6333.

nginx.nix:
  - Three virtual hosts, all on 127.0.0.1, all with WebSocket proxying:
    * localhost:3000  → Grafana listener :3001  (grafana.localhost-proxy)
    * dev.localhost:8082 → localhost:80          (dev.localhost-proxy)
    * localhost:8084  → localhost:80             (localhost-8084-proxy)
  - All loopback-only; the firewall opens no extra ports (allowedTCPPorts = []).
    Port 8081 is intentionally left free for OpsWarden's `client_web` Compose
    service.

5. BINARY SCRIPTS (config/bin/)
--------------------------------------------------------------------------------
Custom logic to fill gaps in the Nix/Wayland ecosystem.

DarkWindow / Hyprchroma:
  - Hyprland visual effect provided directly by the plugin layer.
  - Logic: Runtime behavior is driven by plugin load + sourced theme config.

hypr-plugins-init:
  - Plugin loader for Hyprland. Handles dynamically linked object (.so) loading.

antigravity:
  - System-to-User bridge. Ensures the user can call project-specific binaries
    without absolute path awareness.

cursor:
  - AppImage handler.
  - Logic: Patches `LD_LIBRARY_PATH` and `NIX_LD_LIBRARY_PATH` to expose Nix 
    runtime libraries to the portable AppImage container.
  - Native Keymap Hack: Redirects Debug symbols to Release builds to fix 
    Keyboard-specific crashes on NixOS.

6. VISUAL IDENTITY (Seaglass Theme)
--------------------------------------------------------------------------------
Accent: Teal (#94E2D5).
Components:
  - Hyprland: rounded borders, dual-border active state (conf/theme.conf).
  - Waybar: teal-accented top bar with workspace/media/clock modules.
  Palette is defined once in lib/palette.nix and rendered per tool by lib/colors.nix.

config/
  In-repo configuration tree (Hyprland/Waybar/Rofi/foot live in the separate
  hyprland-config repo, pulled as a flake input):
  bin/, conky/, doom/, fastfetch/, grafana/, icons/, nvim/

config/bin/
  Custom scripts: antigravity, antigravity-ide, cursor, cursor-toggle, devin,
  grafana-generate, grafana-snapshot-sync, hypr-metrics, legion-pulse,
  legion-toggle, nix-flamegraph.py, nix-metrics, rebuild.
  (Hyprland/Waybar helper scripts — hypr-plugins-init, waybar-toggle,
  scss-compile, edex-toggle, etc. — live in hyprland-config/bin/.)

antigravity (home.file.".local/bin/antigravity")
  Wrapper script that runs /etc/nixos/config/bin/antigravity. Lets you invoke the system binary from the user profile.

cursor (home.file.".local/bin/cursor")
  Symlink outside store to /etc/nixos/config/bin/cursor (Cursor editor).

DarkWindow / Hyprchroma
  Active runtime is provided directly by the Hyprland plugin and its sourced config.
  No user-level dw-daemon service is currently defined in the active Nix configuration.
  Legacy dw-* helper scripts were removed from the default runtime path.

assets/
  Root folder: media referenced by the shipped README (logo, screenshots, wallpaper, hero-video).

diagrams/
  Diagram workspace. puml/ contains PlantUML sources, carbon/ contains the
  TreeView HTML renderer and generated HTML views, png/ contains committed
  PNG renders referenced by README and wiki pages.

7. Display, desktop and theme
--------------------------------------------------------------------------------
GDM (services.displayManager.gdm)
  Display manager. Allows choosing Hyprland or GNOME session.

gnome (services.desktopManager.gnome)
  GNOME desktop enabled.

xserver (services.xserver)
  enable = true, xkb.layout = "fr".

xdg.portal
  enable, xdgOpenUsePortal, extraPortals: xdg-desktop-portal-hyprland, xdg-desktop-portal-gtk.

Hyprland (programs.hyprland)
  enable = true, xwayland.enable = true. Config in hyprland-config repo (hyprland.conf, conf/*.conf, waybar/).

Hyprchroma fork / hypr-darkwindow
  RomeoCavazza/Hyprchroma fork compiled into libhypr-darkwindow.so. Used for inactive-window tint and workspace-transition smoothing.

hyprspace (flake input)
  RomeoCavazza/Hyprspace fork locked as a flake input. Used for the overview plugin on Hyprland v0.55.4.

seaglass
  Main visual theme (accent #94E2D5). Files hyprland-config conf/theme.conf.

Waybar (hyprland-config waybar/)
  Status bar (config.jsonc, generated style.css from style.scss (+ injected _variables.scss), scripts).

WaybarCava.sh (hyprland-config waybar/WaybarCava.sh)
  Cava integration script. Generates temporary config in /tmp/bar_cava_config.
  Processes raw ASCII output from Cava into bar characters for Waybar.
  Handles silence masking. Deployed to ~/.config/waybar/ via the Sass
  compilation derivation (waybarConfig in home/tco/hyprland/waybar.nix).

activeapp.sh (hyprland-config waybar/activeapp.sh)
  Active window tracker. Uses `hyprctl activewindow` and `jq` to extract
  class/title. Maps class to Nerd Font icons (firefox -> , code -> 󰨞, etc.).
  Outputs JSON for Waybar. Deployed alongside WaybarCava.sh via waybarConfig.

rofi (hyprland-config rofi/, home.file.".hyprland-config rofi/")
  Active runtime: config.rasi, theme.rasi, custom/column-tco.rasi, scripts/rofi-push.sh, bin/rofi-grid.sh, themes/apps-grid.rasi. Referenced in hyprland.conf ($menu, $powermenu -> rofi-push.sh).

rofi-push.sh (hyprland-config rofi/scripts/rofi-push.sh)
  Dynamic gap management script. Calculates LEFT_GAP = BASE_GAP + 400.
  Sets Hyprland gaps_out to "LEFT,BASE,BASE,BASE" to make room for Rofi
  without overlapping windows. Restores original gaps on exit.

rofi-grid.sh (hyprland-config rofi/bin/rofi-grid.sh)
  Display management. Temporarily increases Hyprland blur size when Rofi
  is active. Kills Waybar on launch and relaunches on exit.

foot (hyprland-config foot/, home.file.".hyprland-config foot/")
  Foot terminal configuration (foot.ini). Theme and colour palette.

GTK (home: gtk)
  theme name = "Adwaita-dark", iconTheme = "Papirus-Dark", cursorTheme = "Bibata-Modern-Ice".

launcher.nix (modules/desktop/launcher/)
  Launcher integration (Rofi etc.).
